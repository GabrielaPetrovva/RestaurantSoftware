# ServeSoftware – MVP

## Цел на продукта

Софтуер за ресторанти, който оптимизира работата на сервитьори, кухня, бар и мениджмънт чрез:

* Бързо приемане и изпращане на поръчки
* Проследяване на статуса в реално време
* Намаляване на грешки
* Прозрачност за мениджърите
* Базови AI прогнози (натоварване, популярност на артикули)
* Възможност клиентът да избира артикули и да генерира QR, който сервитьорът сканира

---

## Потребителски роли

* **Собственик** – управление на меню, служители, маси и роли, преглед на всички поръчки и метрики
* **Мениджър** – управление на служители и меню, преглед на активните поръчки, наблюдение в реално време
* **Сервитьор** – обработка на поръчки, следене на статусите на масите, получаване на известия
* **Готвач / Барман** – обработка на поръчки и маркиране на статуси
* **Клиент** – разглеждане на меню и генериране на QR код за поръчка

---

## Основен User Flow

### Клиент

1. Отваря уеб страницата с менюто
2. Избира ястия
3. Генерира QR код
4. Показва QR на сервитьор

### Сервитьор

1. Сканира QR кода и получава списъка с артикули
2. Потвърждава поръчката и избира маса
3. Поръчката се записва в базата данни
4. Новата поръчка автоматично се появява в кухнята/бара

### Готвач / Барман

1. Преглежда новите поръчки
2. Изпълнява поръчките
3. Маркира ги като „в процес“ или „готово“

### Управител

1. Наблюдава процеса на работа
2. Следи натовареността на персонала

## Backend (Firebase)

### Архитектура

* Firestore – база данни
* Firebase Auth – логин за служители
* Cloud Functions – бизнес логика и защитени операции
* Firebase Storage – PDF сметки
* Security Rules – контрол на ролите
* Realtime listeners – live обновяване за кухня, бар и сервитьори

### Основни Backend функционалности

* Управление на меню, маси, служители и роли
* Създаване и обработка на поръчки
* Live статуси за кухня, бар, сервитьори и мениджъри
* Клиентски избор → QR → сервитьор сканира → поръчка
* Генериране на PDF сметки
* Базови AI прогнози чрез външно API или прост алгоритъм

### Структура на базата данни (Firestore)

#### Колекции

* `menus` – меню на ресторанта
* `menu_categories` – категории
* `tables` – маси и статуси
* `employees` – служители + роли
* `carts` – клиентски избори преди създаване на поръчка
* `orders` – поръчки + артикули + статуси
* `analytics` – метрики и прогнози

#### Примерен документ **orders**

```json
{
  "orderId": "ord001",
  "tableId": "T05",
  "waiterId": "emp123",
  "items": [
    { "id": "burger01", "qty": 2, "type": "kitchen", "status": "pending" },
    { "id": "cola02", "qty": 1, "type": "bar", "status": "pending" }
  ],
  "createdAt": 1702030443000,
  "status": "new"
}
```

### Cloud Functions (минимум)

* `convertCartToOrder` – превръща клиентски cart в поръчка
* `generatePdfBill` – генерира PDF сметки
* `onOrderCreated` – обновява кухня/бар в реално време
* `cleanupExpiredCarts` – изтрива стари carts
* `aiPredictMetrics` – базови прогнози

### Security Rules (пример)

```javascript
match /orders/{orderId} {
  allow read: if request.auth != null;
  allow write: if request.auth.token.role in ["waiter", "manager", "owner"];
}

match /menus/{menuId} {
  allow write: if request.auth.token.role in ["manager", "owner"];
  allow read: if true;
}
```

## QR Workflow

* Клиент избира артикули → генерира се QR код с `cartId`
* Сервитьор сканира QR → Cloud Function създава поръчка
* Поръчката се появява мигновено в кухня/бар

### Пример JSON QR:

```json
{ "cartId": "cart998" }
```

---

## AI Прогнози (базови)

* **Натовареност**: вход – история на поръчките за последните 60 минути
* **Популярност на артикули**: вход – брой поръчки за последните 7 дни; изход – графика ТОП 5 артикули
* **Средно време за приготвяне**: вход – исторически времена; изход – препоръки за оптимизация

## Edge Cases

* Клиент генерира QR и напуска → cart изтича (cleanupExpiredCarts)
* Сервитьор сканира QR без интернет → локално кеширане / retry
* Две поръчки за една маса в рамките на 10 секунди → atomic write
* Клиент добавя нови артикули след генериран QR → генерира се нов cart

* Live update: < 500 ms
* Генериране на PDF: < 3 сек
* Създаване на поръчка: < 200 ms
